# Ontology Requirements Specification Document (ORSD)

## Goal & Purpose
* **Goal:** To represent [Dialogue Structural/Pragmatic] attributes.
* **Purpose:** This ontology is being developed as part of a PhD project to [analyze discourse / automate annotation].

## Scope & Boundaries
* **Domain:** [e.g., Task-oriented dialogue, Social Media Discourse].
* **Granularity:** [e.g., Utterance level, Turn level].
* **In-Scope:** Speech acts, Gricean maxims.
* **Out-of-Scope:** Acoustic features, sentiment analysis.

## Target Audience
* **Primary:** Researchers in Computational Linguistics.
* **Secondary:** NLP pipelines using Knowledge Graphs.

## Requirements Specification
### Non-Functional Requirements
* **NFR-1:** Implementation in OWL 2.
* **NFR-2:** Metadata must follow DCAT or VoID standards.
* **NFR-3:** Naming convention: CamelCase for classes, lowerCamelCase for properties.

## Knowledge Resources to Reuse
* **Top level ontology:** Semanticscience Integrated Ontology (SIO)
* **Domain Ontologies:** Time Ontology in OWL (owl-time).
